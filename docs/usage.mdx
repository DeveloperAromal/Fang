# Usage

Fang operates through a simple interactive CLI. You type an objective in plain English, and Fang handles the rest.

---

## Starting Fang

```bash
python main.py
```

You'll see the Fang banner followed by an interactive prompt:

```
  ___
 |  _|__ _ _ __  __ _
 | |/ _` | '_ \/ _` |
 |_|\__,_|_| |_\__, |
               |___/

[*] Fang ready. Enter your objective below.
>
```

---

## Writing Prompts

Prompts are plain English descriptions of what you want to find out. You don't need to know tool names or flags. The Planner reads your intent and selects the right tools automatically.

### Tips for effective prompts

**Be specific about what you want to find.** The more detail you give, the better the Planner can select the right tools.

```
# Vague — Planner has to guess
> scan example.com

# Better — Planner knows exactly which tools to use
> scan example.com for open ports, service versions, and web technologies
```

**Include the target explicitly.** Always include the domain or URL in your prompt.

```
> full recon on testfire.net
> check https://target.com for vulnerabilities
> enumerate subdomains of example.com and check for exposed admin panels
```

**Describe the goal, not the method.** You don't need to say "run scan_ports then grab_banners" — just say what you want to learn.

```
# You don't need to do this
> run port scan and banner grabbing on example.com

# This is cleaner and works just as well
> find what services are running on example.com and their versions
```

## Example Prompts by Use Case

### Quick tech stack check
```
> scan https://target.com for technologies and frameworks
```
Selects: `fingerprint_tech`

### Full network exposure assessment
```
> scan target.com for open ports and service versions
```
Selects: `scan_ports → grab_banners`

### Web surface mapping
```
> crawl https://target.com and find all endpoints and hidden paths
```
Selects: `parse_robots`, `crawl_urls`

### Subdomain discovery
```
> enumerate subdomains of example.com
```
Selects: `enumerate_subdomains`

### Passive OSINT
```
> gather domain details and social media presence for example.com
```
Selects: `domain_details`, `social_media_osint`

### Full reconnaissance
```
> full recon on testfire.net
```
or
```
> scan testfire.net for open ports, technologies, and any potential vulnerabilities
```
Selects: all relevant tools in dependency order

## A Complete Session Walkthrough

```
> scan testfire.net for open ports, technologies, and any potential vulnerabilities
```

**Planner output:**
```
[*] Target: https://testfire.net
[*] Planning tool execution...
[*] Selected tools: scan_ports, grab_banners, fingerprint_tech, parse_robots, crawl_urls, scrape_web
[*] Execution order resolved.
```

**Orchestrator runs the pipeline:**
```
[+] scan_ports completed
[+] grab_banners completed
[+] fingerprint_tech completed
[+] parse_robots completed
[+] crawl_urls completed
[+] scrape_web completed
```

**Analyzer processes findings:**
```
[*] Analyzing findings...
[+] Analysis complete
```

**Report is generated:**
```
[*] Generating report...
[+] Report saved -> reports/report_20250309_142301.md
```

## Viewing Reports

Reports are saved to the `reports/` directory. Open them in any Markdown viewer:

```bash
# View in terminal
cat reports/report_20250309_142301.md

# Open in VS Code
code reports/report_20250309_142301.md
```

See [Understanding Reports](/reports) for a detailed guide to the report format and how to interpret findings.

## Exiting

```
> exit
```

## Running Multiple Scans

Each prompt starts a fresh pipeline. Storage is cleared between runs. You can run as many scans as you want in a single session:

```
> scan example.com for subdomains
[scan runs...]
> scan anothertarget.com for open ports
[scan runs...]
> exit
```

<Callout type="warning">
  Each new scan replaces the previous session's findings in memory. If you need to keep the raw output, save the report before starting a new scan — reports are always saved to disk automatically.
</Callout>
